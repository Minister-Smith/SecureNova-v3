第二阶段：计算机网络

Phase 02

目标：深入理解网络协议栈，掌握抓包分析技能

OSI七层模型 vs TCP/IP四层模型

OSI七层模型
7. 应用层
HTTP、FTP、DNS、SMTP
6. 表示层
数据加密、压缩、编码
5. 会话层
建立、管理会话
4. 传输层
TCP、UDP
3. 网络层
IP、路由
2. 数据链路层
以太网、MAC
1. 物理层
网线、光纤
TCP/IP四层模型
4. 应用层
HTTP、FTP、DNS（对应OSI 5-7层）
3. 传输层
TCP、UDP
2. 网际层
IP、ICMP、ARP
1. 网络接口层
以太网、PPP

以太网与ARP协议

以太网帧结构
目的MAC + 源MAC + 类型 + 数据 + FCS
MAC地址
48位物理地址，网卡唯一标识
ARP协议
地址解析协议，IP→MAC映射
ARP缓存
存储IP与MAC的对应关系

IP协议详解

IP头结构
版本(4位) + IHL(4位) + TOS(8位) + 总长度(16位)
IP分片
当数据包超过MTU时进行分片传输
TTL
生存时间，防止数据包无限循环
路由原理
路由器根据路由表选择最佳路径转发数据包

ICMP协议

ping命令原理
发送ICMP Echo Request，接收Echo Reply
用于测试网络连通性和延迟
traceroute原理
通过递增TTL值追踪数据包路径
每个路由器返回ICMP超时消息

TCP协议详解

三次握手
SYN → SYN+ACK → ACK
四次挥手
FIN → ACK → FIN → ACK
流量控制
滑动窗口机制，防止发送过快
拥塞控制
慢启动、拥塞避免、快速重传

UDP协议

无连接
不需要建立连接，直接发送
不可靠
不保证数据到达，无重传机制
低延迟
适合实时应用：视频、语音、游戏

DNS协议

解析流程
浏览器→本地DNS→根DNS→顶级DNS→权威DNS
递归查询
DNS服务器负责查询直到得到结果
迭代查询
DNS服务器返回下一个服务器地址
DNS缓存
减少重复查询，提高效率

Wireshark抓包分析

捕获过滤器
tcp port 80, ip host 192.168.1.1
显示过滤器
http.request, tcp.flags.syn
追踪TCP流
右键→Follow→TCP Stream
常见协议分析
HTTP、DNS、TCP、UDP、ARP
